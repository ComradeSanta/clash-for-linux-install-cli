# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

This is a Clash (mihomo) proxy kernel installer and manager for Linux. It provides a `clashctl` command to manage the kernel lifecycle, subscriptions, and configuration. The dashboard is a built-in web UI served by the kernel itself (no separate frontend project) — defaults to `http://<ext-addr>:<ext-port>/ui`, with a public mirror at `URL_CLASH_UI` from `.env`.

## Common Commands

```bash
# Install (kernel_name: mihomo or clash, subscription_url: optional)
bash install.sh [mihomo|clash] [http://subscription-url]

# Uninstall (also cleans up service unit and RC file hooks)
bash uninstall.sh

# Lint shell scripts
shellcheck scripts/*.sh scripts/cmd/*.sh scripts/init/*.sh

# Quick validation after changes
bash -n scripts/cmd/clashctl.sh  # syntax check
shellcheck scripts/cmd/clashctl.sh
```

## Architecture

### Entry Points
- `install.sh` - Sources `clashctl.sh` + `preflight.sh`, then drives a fixed sequence: `_valid` → `_parse_args` → `_prepare_zip` → `_detect_init` → `cp -rf . "$CLASH_BASE_DIR"` → `_set_envs` → `_install_service` → `_apply_rc` → `_merge_config` → `_detect_proxy_port` → `clashui`/`clashsecret`/`clashtun off; clashtun on`. Ends by calling `clashctl` inside the user's shell.
- `uninstall.sh` - Stops the kernel (with Tun-off guard if needed), uninstalls service unit, removes RC-file hooks, drops the crontab entry, deletes `$CLASH_BASE_DIR`.

### Core Scripts
- `scripts/preflight.sh` - Preflight checks, init system detection (systemd/OpenRC/runit/SysVinit), service file installation (substitutes `placeholder_*` patterns), RC file setup, subscription download. Sets `service_*` arrays that `clashctl.sh` consumes.
- `scripts/cmd/clashctl.sh` - All `clashctl` subcommands: `on/off/status/ui/tun/mixin/sub/secret/upgrade/node/proxy`. Self-contained with functions sourced into the user's shell via the RC file.
- `scripts/cmd/common.sh` - Shared utilities: `_okcat`/`_failcat` (colored output), `_download_config`/`_download_convert_config`, `_valid_config` (runs `$BIN_KERNEL -t` to validate), `_detect_ext_addr`, `_get_random_port`, `_is_port_used` (`ss`/`netstat` fallback), and config path resolution from `.env`.
- `scripts/cmd/clashctl.fish` - Fish-shell wrapper that delegates to the bash functions via `bash -i -c`.

### Init System Adapters
- `scripts/init/systemd.sh`, `OpenRC.sh`, `runit.sh`, `SysVinit.sh` - Service definition templates using `placeholder_*` tokens.
- `preflight._detect_init` picks an init by reading `/proc/1/exe`, falling back to `nohup` inside containers (detected via cgroup) or when running as non-root. Containers and no-service-manager installs get a nohup wrapper that fully detaches stdio.

### Configuration & Data Files
- `.env` - Lives in the source repo. Read by `common.sh` at runtime via a relative path. The source copy is the **input** to `install.sh`; after install the script also writes the resolved `INIT_TYPE`/`KERNEL_NAME`/`CLASH_BASE_DIR`/`VERSION_MIHOMO` back via `_set_env` — but to the **installed** copy at `$CLASH_BASE_DIR/.env`, not the source. Edit the source `.env` and re-run `install.sh` to change install options.
- `resources/mixin.yaml` - User mixin config. Survives subscription updates. Supports deep-merge with prefix/suffix/override/inject strategies. Also stores `._custom.*` for script state (e.g. `system-proxy.enable`).
- `resources/profiles.yaml` - Subscription metadata (`id`, `path`, `url`, `use`). Managed by `clashsub` commands.
- `resources/profiles/` - Individual subscription config files, one per profile id.
- `resources/dist/` - Web UI assets served by the kernel.
- `resources/dist/` also stores downloaded `mihomo`/`clash`/`yq`/`subconverter` binaries.
- `resources/*.mmdb`, `resources/*.dat` - GeoIP and geosite databases.

### Config File Hierarchy
Three config files feed the running kernel:
1. `CLASH_CONFIG_BASE` - Original subscription config (overwritten on `clashsub use`).
2. `CLASH_CONFIG_MIXIN` - User mixin overrides (`resources/mixin.yaml`).
3. `CLASH_CONFIG_RUNTIME` - Merged config actually used by the kernel (regenerated by `_merge_config`).

### Config Merging (`_merge_config` in `clashctl.sh`)
Single `yq eval-all` expression that deep-merges `$CLASH_CONFIG_BASE` with `$CLASH_CONFIG_MIXIN`, then applies special key handling:
- `rules.prefix/suffix` - Prepended/appended to subscription rules.
- `proxies.prefix/suffix` - Prepended/appended to subscription proxies.
- `proxies.override` - Replace by `name`.
- `proxy-groups.prefix/suffix` - Prepended/appended to groups.
- `proxy-groups.override` - Replace by `name`.
- `proxy-groups.inject` - Non-invasive injection: adds proxy names to an existing group's `.proxies` list (auto-deduplicated). Useful for chain proxies without overriding the whole group.
- `._custom` - Stripped from mixin before merge; used purely as script-state sidecar.
- On invalid output, `_valid_config` is rerun and the previous `CLASH_CONFIG_RUNTIME` is restored from `CLASH_CONFIG_TEMP` (atomic-ish rollback).

### Node Management Pattern
Node operations (`clashnode list/test/use`) use the kernel's REST API at `http://${EXT_IP}:${EXT_PORT}` with Bearer token auth (from `_get_secret`). This is a runtime-only API — it does not persist to config files. `_urlencode` shells out to `python3` with a `od`-based fallback.

### Subscription Profile System
- Profiles tracked in `profiles.yaml` with sequential numeric IDs assigned by `_sub_add` (`max(existing ids) + 1`).
- `clashsub add` downloads, validates (raw → fallback to subconverter), and stores at `profiles/<id>.yaml`.
- `clashsub use <id>` copies that profile to `CLASH_CONFIG_BASE` and triggers `_merge_config_restart`.
- `clashsub update [id]` re-downloads; if the updated profile is the active one (`use == id`), it auto-reapplies via `clashsub use`.
- `clashsub update --auto` registers a crontab line running every 2 days; `--convert` forces the subconverter path.

### Service-Agnostic Commands (Placeholder Pattern)
`preflight._detect_init` populates `service_*` arrays; `preflight._install_service` then runs a python3 inline script to substitute `placeholder_*` tokens in `clashctl.sh` + `common.sh` (chosen over sed for cleaner multi-token replacement). Tokens replaced:
```bash
placeholder_start / placeholder_sudo_start / placeholder_stop / placeholder_status
placeholder_is_active / placeholder_log / placeholder_follow_log / placeholder_watch_proxy
placeholder_cmd_path / placeholder_cmd_args / placeholder_cmd_full
placeholder_log_file / placeholder_pid_file / placeholder_kernel_name / placeholder_kernel_desc
```
The init-system service file also gets `placeholder_*` substitution via `sed -i`.

## Key Details

- **`yq` is required** for all config manipulation — shell scripts do not parse YAML directly.
- **Tun mode** requires sudo because the kernel needs `CAP_NET_ADMIN` for TUN device creation. The scripts handle start/stop with elevated privileges and restore terminal state with `stty opost` after TUN toggles. On startup failure with the `unsupported kernel version` message, mihomo sets `.tun.auto-redirect = false` and retries; if that also fails, `_tunoff` runs and the script aborts.
- **Port conflict detection** runs in `_detect_proxy_port` on `clashon` and in `_detect_ext_addr`/`_detect_subconverter_port` elsewhere — if a configured port is in use and the kernel isn't running, a random port is assigned and written back to the mixin.
- `_set_system_proxy` reads `mixed-port`/`port`/`socks-port` and `authentication[0]` from the runtime config, plus `bind-address`/`allow-lan` for the bind address. The bind is `127.0.0.1` unless `allow-lan: true`.
- The `watch_proxy` function (sourced via RC) auto-runs `clashon` on interactive shell start — disabled by default in the generated RC, uncomment `watch_proxy` to enable.
- `.env` paths are resolved inside `common.sh` — all scripts source `common.sh` to get config locations.
- The fish wrapper (`clashctl.fish`) is a thin shim that calls `bash -i -c '<func> "$@"'` for each command.

## Tun Mode Caveats

The Tun mode implementation has a known pattern: when stopping the kernel with Tun active, `_tunoff` is called before `_tunoff` returns, and `clashoff` guards the same path. Terminal issues from TUN toggles are handled with `stty opost` restoration. The `auto-redirect` fallback is disabled for unsupported kernel versions.

## Shellcheck Exclusions (`.shellcheckrc`)
- `SC1091` - Not following sourced files (`clashctl.sh`/`common.sh`/`fish` wrapper don't get re-lint-traversed).
- `SC2155` - Declare and assign separately (intentional in this codebase).
- `SC2296` - Parameter expansion in `${var}` context (used for config key expansion).
- `SC2153` - Possible misspelled output var (from `yq` queries).
